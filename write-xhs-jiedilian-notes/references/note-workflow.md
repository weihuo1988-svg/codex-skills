# 小红书姐弟恋笔记工作流

## 1. 输入检查

当用户调用本 skill 时，先确认是否已有一张或多张爆款文章截图。

- 创作前必须读取账号定位文件。默认路径：`/Users/alinyao/Vicky/小红书笔记/姐弟恋赛道/账号定位.md`。如果用户给出新的账号定位文件或目标文件夹里存在更近的 `账号定位.md`，优先读取更近的文件。
- 如果没有截图，要求用户提供截图。
- 如果截图文字不清晰，说明哪些区域无法识别，并基于可见内容先做草稿或请求更清晰截图。
- 如果用户已经给出目标人设、语气、产品目的或保存文件夹，把这些约束纳入草稿。
- 不在用户确认前写入最终文件夹。

## 2. 账号定位校准

读取 `账号定位.md` 后，先完成内部校准，不需要把这部分完整输出为正文：

- 判断本篇属于哪类内容：主理人日常、投稿/会客厅故事、女性成长/关系方法论，或混合型。
- 把每篇笔记放回「年下恋姐姐会客厅」的账号心智：可以写甜，但要帮姐姐把关系想明白。
- 让正文背后始终有女性独立清醒的价值观：姐姐可以心动，但不替对方长大；可以主动，但不失去边界；可以享受恋爱，但保留自己的节奏和价值。
- 不要直接输出定位口号，不要把正文写成「女性独立清醒」的宣言或说教。用具体场景、细节选择、情绪转折和结尾问题承载观点。
- 每篇设置一个可讨论的具体矛盾，例如年龄差、婚育节奏、成熟度、金钱观、父母态度、姐姐是否承担过多、弟弟是真爱还是依赖。
- 结尾给一个轻量互动问题，让读者想评论，而不是只写泛泛的「欢迎评论」。

如果本篇是「主理人日常」：

- 图片里的姐姐形象、弟弟形象、出生年份或年龄差信息要和既有账号设定保持一致。
- 如果 `账号定位.md` 或已确认素材中已有固定男女主参考，生成图片前先沿用；如果没有明确参考，向用户确认，不要临时随机换人设。
- 不要把每篇主理人日常都写成完美恋爱赢家，要保留真实犹豫、边界和成长感。

如果本篇是「投稿/会客厅故事」：

- 可以不固定男女主外貌，更适合弱化脸部或使用情绪场景。
- 文案中可自然说明是投稿、身边姐姐的故事、会客厅来信，避免让读者误以为全是主理人亲身经历。
- 注意隐私，不写真实姓名、公司、学校、住址、联系方式等可识别信息。

## 3. 爆款截图拆解

对每张截图做轻量拆解，重点提取可复用模式：

- 标题：钩子词、情绪强度、悬念、利益点、目标读者。
- 开头：第一句话如何抓住「姐姐」或「年下恋」人群。
- 正文结构：冲突、反差、具体细节、情绪转折、观点输出、评论引导。
- 语气：亲密倾诉、反思复盘、清醒建议、甜蜜日常、轻微吐槽等。
- 排版：短句、分段、列表、金句密度、是否使用 emoji。
- 标签：赛道标签、关系标签、情绪标签、场景标签的组合。
- 图片：人物年龄感、氛围、构图、色调、场景、是否有文字封面感；特别观察是否在两个人附近标注出生年份或年龄。

不要把截图原文完整转写给用户。只总结结构和少量必要短语，避免直接复制作权文本。

## 4. 原创复刻规则

复刻「爆款结构、节奏、情绪推进和标题公式」，不要逐字照抄。

写作默认方向：

- 主题聚焦：姐弟恋、年下恋、姐姐视角、成熟关系、年龄差带来的反差和安全感。
- 人设默认：双方均为成年人；如需具体年龄，女性可写 28-35 岁，男性可写 22-29 岁。
- 内容尺度：暧昧、甜蜜、拉扯、反思可以写；不要写未成年人、露骨性描写、操控或PUA合理化。
- 语言风格：像真实小红书用户，不像营销稿；多用具体场景、动作和心理活动。
- 观点底色：围绕账号定位输出稳定观点，但藏在故事推进里；不要把定位文字原样写进正文。
- 话题性：每篇至少有一个让读者能站队、共鸣或补充经历的问题。
- 原创要求：不要复制截图里的独特句子、完整段落、专有表达或个人经历细节。

## 5. 标准草稿格式

先输出草稿给用户确认，使用以下结构：

```markdown
图片：
（展示已生成图片路径，或给出图片生成提示词）

标题：
...

正文：
...

标签：
#姐弟恋 #年下恋 ...

参考拆解：
- 这版主要复刻了：标题钩子 / 开头方式 / 情绪结构 / 标签策略
- 定位校准：内容支柱 / 隐性观点 / 互动问题

待确认：
确认后我会保存到：<目标文件夹或“等待你指定”>
```

正文建议 300-800 字，短段落，每段 1-3 句。除非截图风格明显依赖 emoji，否则少用 emoji。

标题建议 12-26 个中文字符，优先包含「姐弟恋」「年下」「姐姐」「他」等赛道词或隐含关系张力。可以给 1 个主标题；只有用户要求时再给备选标题。

正文结尾优先用具体问题引导互动，例如：

- 如果你是这个姐姐，你会继续等吗？
- 你们觉得这是少年感，还是没责任感？
- 姐姐在年下恋里，到底该不该主动推进未来？
- 年下恋里哪一刻会让你突然清醒？

标签建议 8-12 个，至少包含 2-4 个赛道强相关标签，例如：

- `#姐弟恋`
- `#年下恋`
- `#恋爱日常`
- `#情感`
- `#姐姐`
- `#年下男友`
- `#恋爱脑清醒指南`
- `#亲密关系`

## 6. 图片生成提示

如果需要生成图片，调用可用的图片生成能力。提示词要让画面服务于小红书封面：

- 画幅：优先 3:4 或 1:1。
- 主体：成年女性与成年男性情侣，年龄差可见但不夸张。
- 主理人日常一致性：如果本篇是账号本人的日常，必须沿用既有姐姐和弟弟的长相、年龄信息、出生年份标签和整体生活感；不要每篇换一对随机情侣。
- 投稿/会客厅故事差异：如果不是主理人日常，可以弱化脸部或改用情绪场景，不必强制同一对男女主。
- 年龄差标签：姐弟恋赛道封面优先在两个人附近标注出生年份或年龄，例如在女性附近标 `93`，男性附近标 `03`；这类标签是吸睛点，能让用户一眼理解年龄差。
- 标签位置、对比度与字号：数字放在人物上方或旁边的背景留白处，不要压在人脸、头发、头顶或身体上；白色数字要避开浅色天空、白墙、强反光玻璃等低对比背景，优先放在绿树、深色建筑、阴影或其他有纹理的中深色背景前；字号参考爆款截图，克制清晰即可，不要做成过大的海报字。
- 场景：咖啡店、街边夜景、地铁口、家居沙发、旅行街拍、雨天撑伞等。
- 氛围：温柔、真实、轻电影感、生活化，不要过度影楼风。
- 如果图片模型无法稳定生成文字，先生成无文字情侣照，再用图像编辑或脚本把 `93` / `03` 之类的白色大号数字加到人物附近。
- 禁止：未成年人、校服暗示、露骨姿势、明显性化、低质网红滤镜。

示例提示词：

```text
一张适合小红书情感笔记封面的真实感照片：一位约33岁的成年女性和一位约23岁的成年男性在城市绿植露台并肩站着，男生微微靠近女生，氛围温柔、有年下恋的亲密感，生活化街拍，暖色自然光，3:4构图。在女性附近标注白色大号数字“93”，在男性附近标注白色大号数字“03”，无logo，非露骨。
```

## 7. 确认与修改

草稿后必须等待用户确认。

- 用户说「确认」「可以」「就这版」「保存」「归档」等，才进入保存。
- 用户要求改标题、改风格、改人设、改尺度或改标签时，先修改草稿，再次请求确认。
- 如果用户确认但没有指定文件夹，先问目标文件夹，不要猜路径。

## 8. 保存归档

确认后优先使用 `scripts/save_note.py`。

推荐流程：

1. 将正文保存为临时 `body.md`。
2. 将标签保存为临时 `tags.txt`，一行一个或空格分隔均可。
3. 准备已生成的图片文件；如果暂时只有图片提示词，将提示词保存为临时 `image_prompt.md`。
4. 运行脚本：

```bash
python3 /Users/alinyao/.codex/skills/write-xhs-jiedilian-notes/scripts/save_note.py \
  --output-dir "/path/to/target-folder" \
  --title "笔记标题" \
  --body-file "/path/to/body.md" \
  --tags-file "/path/to/tags.txt" \
  --image "/path/to/cover.png" \
  --source-screenshot "/path/to/screenshot1.png"
```

如果只有图片提示词，使用：

```bash
python3 /Users/alinyao/.codex/skills/write-xhs-jiedilian-notes/scripts/save_note.py \
  --output-dir "/path/to/target-folder" \
  --title "笔记标题" \
  --body-file "/path/to/body.md" \
  --tags-file "/path/to/tags.txt" \
  --image-prompt-file "/path/to/image_prompt.md"
```

脚本会创建形如 `20260705-213000-xhs-jiedilian-note/` 的文件夹，并写入：

- `note.md`
- `cover.<ext>`，如提供图片
- `image_prompt.md`，如提供图片提示词

如果用户指定了不同命名规则或文件格式，按用户指定保存，但仍保留标题、正文、标签、图片或图片提示词、源截图信息。
