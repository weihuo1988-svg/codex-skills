# 小红书亲子赛道笔记工作流

## 1. 输入检查

当用户调用本 skill 时，先确认是否已有一张或多张爆款文章截图，或一个足够清晰的亲子选题。

- 创作前必须读取账号定位文件。默认路径：`/Users/alinyao/Vicky/小红书笔记/亲子赛道/账号定位.md`。如果用户给出新的账号定位文件或目标文件夹里存在更近的 `账号定位.md`，优先读取更近的文件。
- 如果没有截图但用户给了明确选题，可以基于账号定位和本 skill 的 house patterns 先做草稿，并在参考拆解里标注 `无参考截图，按账号定位生成`。
- 如果截图文字不清晰，说明哪些区域无法识别，并基于可见内容先做草稿或请求更清晰截图。
- 如果用户已经给出目标人设、语气、产品目的、孩子年龄、内容禁区或保存文件夹，把这些约束纳入草稿。
- 不在用户确认前写入最终文件夹。

## 2. 账号定位校准

读取 `账号定位.md` 后，先完成内部校准，不需要把这部分完整输出为正文：

- 判断本篇属于哪类内容：主理人育儿日记、离异带娃生活片段、育儿经验/方法论、亲子关系讨论、妈妈成长/边界感、投稿/妈妈来信，或混合型。
- 把每篇笔记放回「离异带娃亲子账号」的心智：可以写累和委屈，但最后要回到妈妈和孩子如何一起变好一点。
- 让正文背后始终有稳定价值观：离异不是污点，孩子不是大人的情绪容器，妈妈也不是只能牺牲的人。
- 不要直接输出定位口号，不要把正文写成「女性成长」或「科学育儿」宣言。用具体场景、细节选择、情绪转折和结尾问题承载观点。
- 每篇设置一个可讨论的具体矛盾，例如孩子突然懂事、爸爸缺席如何解释、妈妈是否要一直扛、孩子发脾气怎么接、重新开始生活的边界、前任参与育儿的尺度。
- 结尾给一个轻量互动问题，让读者想评论，而不是只写泛泛的「欢迎评论」。

如果本篇是「主理人育儿日记」：

- 只沿用账号定位里已经确认的亲子设定，不要临时发明孩子姓名、年龄、学校、城市、抚养权细节、前任信息或家庭收入。
- 如果缺少必要设定，先用模糊表达，例如 `孩子`, `小朋友`, `那天晚上`, `家长会`, 不要硬填私人细节。
- 不要把每篇日记都写成苦尽甘来或完美妈妈，要保留真实犹豫、复盘和成长感。

如果本篇是「投稿/妈妈来信」：

- 可以写成 `有位妈妈留言`, `一个读者妈妈说`, `妈妈来信`，避免让读者误以为全是主理人亲身经历。
- 必须匿名化真实姓名、学校、公司、住址、联系方式、医院、车牌、具体班级和过度可识别的家庭事件。
- 不把投稿里的个案上升为普遍判断，结尾可以邀请读者补充自己的做法。

## 3. 爆款截图拆解

对每张截图做轻量拆解，重点提取可复用模式：

- 标题：钩子词、情绪强度、悬念、利益点、目标读者。
- 开头：第一句话如何抓住妈妈、单亲妈妈、离异带娃家长或亲子关系人群。
- 正文结构：冲突、反差、具体细节、情绪转折、观点输出、评论引导。
- 语气：亲密倾诉、真实日记、反思复盘、温柔建议、轻微吐槽、讨论发问等。
- 排版：短句、分段、列表、金句密度、是否使用 emoji。
- 标签：赛道标签、育儿标签、关系标签、情绪标签、场景标签的组合。
- 图片：亲子关系是否一眼可见、孩子是否可识别、场景是否生活化、封面文字是否短而清晰。

不要把截图原文完整转写给用户。只总结结构和少量必要短语，避免直接复制作权文本。

## 4. 原创复刻规则

复刻「爆款结构、节奏、情绪推进和标题公式」，不要逐字照抄。

写作默认方向：

- 主题聚焦：离异带娃、单亲妈妈、育儿日记、亲子关系、妈妈成长、生活复盘、育儿经验讨论。
- 内容尺度：可以写疲惫、愧疚、破防、重建、边界、亲子冲突；不要写儿童隐私、猎奇冲突、未证实诊断、法律细节或对前任的公开审判。
- 语言风格：像真实小红书用户，不像教育号或营销号；多用具体场景、动作和心理活动。
- 观点底色：围绕账号定位输出稳定观点，但藏在故事推进里；不要把定位文字原样写进正文。
- 话题性：每篇至少有一个让读者能站队、共鸣或补充经历的问题。
- 原创要求：不要复制截图里的独特句子、完整段落、专有表达或个人经历细节。

## 5. 标准草稿格式

先输出草稿给用户确认，使用以下结构：

```markdown
图片：
（展示已生成图片路径，或给出图片生成提示词）

标题：
...

正文：
...

标签：
#离异带娃 #亲子关系 ...

参考拆解：
- 这版主要复刻了：标题钩子 / 开头方式 / 情绪结构 / 标签策略
- 定位校准：内容支柱 / 隐性观点 / 互动问题 / 隐私处理

待确认：
确认后我会保存到：<目标文件夹或“等待你指定”>
```

正文建议 300-800 字，短段落，每段 1-3 句。除非截图风格明显依赖 emoji，否则少用 emoji。

标题建议 12-28 个中文字符，优先包含「离异带娃」「单亲妈妈」「一个人带娃」「孩子」「妈妈」「育儿」等赛道词或具体场景。可以给 1 个主标题；只有用户要求时再给备选标题。

正文结尾优先用具体问题引导互动，例如：

- 孩子突然懂事，你会心疼还是会提醒自己别让他替大人扛？
- 离异后，要不要主动跟孩子解释爸爸的缺席？
- 孩子发脾气那一刻，你会先抱他，还是先讲道理？
- 单亲妈妈最难的，是钱、时间，还是没有人兜底？

标签建议 8-12 个，至少包含 2-4 个赛道强相关标签，例如：

- `#离异带娃`
- `#单亲妈妈`
- `#亲子关系`
- `#育儿日记`
- `#育儿经验`
- `#带娃日常`
- `#妈妈成长`
- `#情绪稳定的妈妈`
- `#女性成长`
- `#亲子沟通`

## 6. 图片生成提示

如果需要生成图片，调用可用的图片生成能力。提示词要让画面服务于小红书封面：

- 画幅：优先 3:4 或 1:1。
- 主体：一位成年妈妈或家长和一个孩子，亲子关系一眼可见；如果是妈妈独处复盘，也要用孩子物件如书包、绘本、饭盒、夜灯、校服外套暗示带娃场景。
- 隐私：默认使用虚构人物或非可识别孩子；优先背影、侧脸、远景、局部动作，不展示真实儿童清晰正脸。
- 场景：家里餐桌、睡前绘本、厨房、楼下便利店、雨天接娃、学校门口但无校名、游乐场、公交/地铁口、夜里收拾书包。
- 氛围：真实、温柔、生活化、有一点情绪张力；不要过度影楼风、苦情大片或灾难化表情。
- 封面字：可用短字卡，如 `离异带娃`, `他那句话`, `妈妈也会累`, `不是我撑住了`。放在背景留白处，不要压在孩子脸、身体、校徽、文件或隐私信息上。
- 禁止：裸露或性化儿童、校徽/校名、真实文件、病历、车牌、门牌、聊天记录、猎奇哭闹特写、法庭或医院细节堆叠。

示例提示词：

```text
一张适合小红书亲子笔记封面的真实感照片：一位约35岁的中国妈妈和一个约7岁的孩子坐在家里餐桌旁，桌上有绘本、饭盒和一盏暖色小台灯，妈妈轻轻低头听孩子说话，孩子以侧背影呈现不可识别，氛围温柔但有一点生活疲惫感，3:4构图，背景干净，无logo、无校名、无真实文件。画面留白处可添加短字“离异带娃”。
```

如果图片模型无法稳定生成文字，先生成无文字亲子生活照，再用图像编辑或脚本把短封面字加到背景留白处。

## 7. 确认与修改

草稿后必须等待用户确认。

- 用户说「确认」「可以」「就这版」「保存」「归档」等，才进入保存。
- 用户要求改标题、改风格、改人设、改尺度、改标签或加强/弱化离异带娃标签时，先修改草稿，再次请求确认。
- 如果用户确认但没有指定文件夹，先问目标文件夹，不要猜路径。

## 8. 保存归档

确认后优先使用 `scripts/save_note.py`。

推荐流程：

1. 将正文保存为临时 `body.md`。
2. 将标签保存为临时 `tags.txt`，一行一个或空格分隔均可。
3. 准备已生成的图片文件；如果暂时只有图片提示词，将提示词保存为临时 `image_prompt.md`。
4. 运行脚本：

```bash
python3 /Users/alinyao/.codex/skills/write-xhs-qinzi-notes/scripts/save_note.py \
  --output-dir "/path/to/target-folder" \
  --title "笔记标题" \
  --body-file "/path/to/body.md" \
  --tags-file "/path/to/tags.txt" \
  --image "/path/to/cover.png" \
  --source-screenshot "/path/to/screenshot1.png"
```

如果只有图片提示词，使用：

```bash
python3 /Users/alinyao/.codex/skills/write-xhs-qinzi-notes/scripts/save_note.py \
  --output-dir "/path/to/target-folder" \
  --title "笔记标题" \
  --body-file "/path/to/body.md" \
  --tags-file "/path/to/tags.txt" \
  --image-prompt-file "/path/to/image_prompt.md"
```

脚本会创建形如 `20260705-213000-xhs-qinzi-note/` 的文件夹，并写入：

- `note.md`
- `cover.<ext>`，如提供图片
- `image_prompt.md`，如提供图片提示词

如果用户指定了不同命名规则或文件格式，按用户指定保存，但仍保留标题、正文、标签、图片或图片提示词、源截图信息。
